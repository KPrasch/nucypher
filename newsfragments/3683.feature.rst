Added ``toTokenBaseUnits`` operator for condition variable operations (takes ``decimals`` as value to convert token amounts to base units, e.g., ``1.5`` with ``decimals=18`` becomes ``1500000000000000000``), useful for ERC-20 token comparisons, and increased condition limits (``SequentialCondition.MAX_NUM_CONDITIONS`` to 20, ``MAX_MULTI_CONDITION_NESTED_LEVEL`` to 4).
